Intro¶

In Project 1, we performed ridge regression on our Concrete Compressive Strength dataset. Despite my best efforts with tuning the model in the last project, I was not able to get ridge regression to substantially and consistently outperform linear regression. There are several possible sources of error I would contribute to this, one being that I had not implemented a lasso. Additionally, the data is not nececarrily dependent on any linear factors. With further research into the nature of data analysis on compositional data (data that representing components adding to a whole), I found I might have been barking up the wrong tree. In the paper Compositional Data and Their Analysis: An Introduction, Vera Pawlowsky-Glahn and Juan Jos Egozcue make several enlightening statements in regards to how compositional data can cause issues when trying to create models without consideration.

"The properties peculiar to compositional data arise from the fact that they represent parts of some whole; therefore, they convey only relative information ... This fact conditions the relationships that variables have to one another - they are not free to vary independently in the way that unconstrained variables can"

There are several ways to approach solving this. In our case, the first thing we are going to do is move away from linear regression, and instead, we will be using a decision tree model. Additionally, instead of only using data from concrete that is 28 days old (like in the last project), I am going to allow age to be an independent variable in these models, as I originally had hoped that filtering it down in the last report would be beneficial in drawing conclusions about the importance of composition, but I beleive that age may play a larger part in compressive strength then I had previously thought

Decision Tree¶

A Decision Tree is a model differing from linear regression. As discussed in the class, it has many benefits over it such as

  • Being able to capture non-linearity
  • Working without having to councern ourselves with scaling
  • Being easily usable or categorizagion or regression

It is for these reasons coupled with some of the drawbacks realized in the last project, that we will be using regression trees.

The best way to describe a decidion tree is as a set of rules. If we decide to look at it like a binary tree, these rules can be represented as nodes, and when we apply a rule we can take a "walk" down the "tree", with the traveled path determined by conditional statements on the paths.

Drawbacks to This Model¶

For Starters, It gets complicated... Really complicated (As evidenced by the graph of the tree above). It can take a lot of time and effort to get a tree that is more balanced than this, as for a dataset this size, a tree this big will usually overfit the data, and sub optimal pruning could end up making the model less accurate on the testing data. To combat this... we are introducing the gradient boosted decision tree.

Gradient Boosted Decision Trees¶

A gradient boosted decision tree can provide a "boost" over our regular decision tree by iteratively fitting new decision trees to the residuals of the previous ones, and essentially building a strong model out of weaker ones. We will use this method to further improve on the error of our decision tree model.

Conclusion¶

So in summary, this project can be thought of as an example of how there are many ways to perform data analysis, and oftentimes, the more forethought is put into the analysis, the more complete a model one will be able to make. In this project I introduced how one could perform simple linear regression on data. The model this resulted in had a RMSE of about 10.6 MPa (Megapascals). To put that into perspective, the mean compressive strength for all concrete samples was 36.75 MPa. This made the average error about 29% of the mean, which (in my opinion) is not accurate enough to use the model to predict the properties of a composition. Because of this we decided to encorporate a new model, the decision tree, which solved our non-linearity issue that comes with modeling compositional data. This model put us in a much closer ball park, with a median RMSE of about 6.19 MPa (16.8% of the mean compressive strengh), a significant improvement over the linear model. Finally, to solve the innate overfitting issues that can come with decision trees, I implemented the gradient boosted decision tree, which iteratively fits the data by combining decision tree models built on mitigating resituals of previous iterations, which resulted in a final RMSE of about 5.07 MPa (13.7% of mean compressive strength). If I were to expand on this project, I would most likely try other methods of improving on the decision tree model. For example, I might try to use random forests or bagging.
